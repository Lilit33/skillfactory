# Оформление Заказа

Lilit Vardanyan



**Цель:** Успешное оформление заказа.

**Акторы:** Пользователь, Система заказа пиццерии, Система платежей.

**Предварительные условия:** Пользователь наполнил корзину и перешел к оформлению заказа.

## **Основной поток:**

1. Пользователь выбирает адрес доставки.

2. Пользователь выбирает способ оплаты:

	- Наличными при получении.

	- Картой при получении заказа.

	- Оплатой картой онлайн во время оформления заказа:

		+ Пользователь вводит данные банковской карты.

		+ Система проверяет валидность оплаты.

		+ Пользователь указывает сумму чаевых (необязательно).

		+ Система отправляет электронный чек на электронную почту пользователя.

3. Пользователь подтверждает заказ.

4. Система отправляет пользователю уведомление о подтверждении заказа.

5. Система отображает информацию о процессе доставки, включая примерное время доставки, местоположение курьера, контакты курьера и статус заказа.
## **Альтернативные потоки:**

1. Если пользователь указал некорректный адрес доставки:

	- В сотрудничестве с курьером определяется правильный адрес.

	- Пользователь вносит изменения в адрес доставки через систему.

2. Если данные банковской карты невалидны или на карте недостаточно средств:

	- Система уведомляет пользователя о проблеме с оплатой.

3. Если пользователь вводит недостаточно или некорректные данные для оплаты:

	- Система указывает, какие данные необходимо добавить или исправить.

4. Если пользователь пытается оформить заказ в нерабочее время пиццерии:

	- Система уведомляет пользователя о времени работы пиццерии.

5. Если пользователь хочет отменить заказ в течение определенного времени после оформления:

	- Система позволяет отменить заказ, если это возможно в соответствии с политикой пиццерии.

**Постусловие:** Заказ успешно оформлен, и пользователь ожидает его доставку.
